Projects

Battery Sequence

Project Overview

A puzzle VR game that was developed in under 7 days for a game jam with the theme was "Single Use". The twist that we introduced was limited inputs. The player is only allowed to press each button once (although there are ways to recharge) in order to solve each puzzle.

My Role

Being the only programmer, i was responsible for the developement all of the mechanics (grabbing, sockets, interactions, etc...). I also played a major role in the game and level design.

Lost In The Open

Project Overview

A 2.5D roguelike that combines procedurally generated biomes, characters and map with a challenging and engaging turn-based combat.

My Role

Responsible for the procedural generation of the world (Hex Tilemap). This was achieved with several rules and algorithms, with perlin noise being the main idea behind the generation.

Go Home

Project Overview

A sandbox game with visual novel and puzzle elements. Go Home is a game where you take on the role of a lost puppy that is searching for it's owner. To do so, you will have to explore, complete quests and solve puzzles.

My Role

Being the team's programmer, i was tasked with implementing everything inside Unity. One of the biggest challenges of this project was ensuring that the player had a smooth and responsive movement.

Dialogue Editor

Project Overview

Whilst working on a personal project, i felt the need to create a more robust dialogue system. As such, i developed a custom dialogue editor. The goal of the editor was to allow the easy creation of complex dialogue trees.

My Role

The whole editor was programmed by me without the use of any third party asset. Applied several principles in order to come up with a dialogue tree editor that allowed the user to create complex trees. For instance, each node can have a message, a sprite change, a name change, special events associated (open shop UI, open door, etc), and more!
